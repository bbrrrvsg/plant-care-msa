# 펌프 제어 로직

## 개요
ESP32가 센서 데이터를 전송하면 서버가 식물 도감의 물주기 정보를 기반으로
토양 수분 임계값과 펌프 가동 시간을 계산하여 응답합니다.

---

## 물주기 조건표

| 물주기 | 토양수분 임계값 | 펌프 가동시간 | 의미 |
|--------|---------------|-------------|------|
| 촉촉하게 유지 | 50% 이하 | 1초 | 조금씩 자주 급수 |
| 표면이 말랐을 때 | 30% 이하 | 3초 | 보통 급수 |
| 대부분 말랐을 때 | 15% 이하 | 5초 | 충분히 급수 |
| 기본값 (알 수 없음) | 30% 이하 | 3초 | 기본 급수 |

---

## 데이터 흐름

```
ESP32 센서 데이터 전송
      ↓
sensor-service
      → deviceId로 plantId 조회 (sensor_device 테이블)
      → plant-service에 plantId로 speciesCode 조회
      → plant-service에 speciesCode로 watering 정보 조회
      → 물주기 텍스트 → 임계값 + 가동시간 계산
      ↓
ESP32 응답
{ "pump": true, "duration": 3000 }
      ↓
ESP32 → duration(ms)만큼 펌프 가동
```

---

## 서버 응답 형식

```json
{
  "pump": true,
  "duration": 3000
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| pump | boolean | 펌프 가동 여부 |
| duration | int | 펌프 가동 시간 (밀리초) |

---

## 주의사항
- 펌프 가동 시간은 화분 크기와 펌프 성능에 따라 조정 필요
- 토양 수분 센서 캘리브레이션 값 (건조: 2420, 습윤: 820) 기준
- 물주기 텍스트가 위 3가지에 해당하지 않으면 기본값 적용
